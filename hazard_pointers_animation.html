üìö How Hazard Pointers Work

Problem: In lock-free data structures, threads access shared nodes concurrently. If we delete a node while another thread is reading it, we get use-after-free bugs!

Solution: Hazard Pointers protect nodes from deletion:

  1. Acquire: Before accessing a node, a thread sets a hazard pointer to it
  2. Protect: The hazard pointer prevents other threads from freeing that node
  3. Retire: When deleting a node, add it to a retire list instead of freeing immediately
  4. Scan: Periodically scan all hazard pointers - only free nodes that aren't protected
  5. Release: When done, clear the hazard pointer to allow eventual deletion

Key Insight: A node can only be freed if NO thread has a hazard pointer pointing to it. This guarantees safe concurrent access without locks!
